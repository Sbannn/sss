### `PriceList`

Представляет одну запись прайс-листа: формат печати и его цену.

```python
@dataclass
class PriceList:
    format: str    # Название формата, например "10x15"
    price: float   # Цена в рублях
```

#### Методы

---

##### `update_price(new_price: float) -> None`

Обновляет цену формата.

| Параметр | Тип | Описание |
|---|---|---|
| `new_price` | `float` | Новая цена в рублях |

**Исключение:** `ValueError` — если `new_price <= 0`.

```python
pl = PriceList("10x15", 15.0)
pl.update_price(20.0)   # OK
pl.update_price(-5.0)   # -> ValueError: Цена должна быть положительной
```

---

##### `get_price() -> float`

Возвращает текущую цену формата.

```python
pl = PriceList("A4", 60.0)
pl.get_price()  # -> 60.0
```

### `Photo`

Фотография, добавляемая в заказ.

```python
@dataclass
class Photo:
    id: str                            # Уникальный идентификатор
    format: str                        # Формат печати ("10x15", "A4" и т.д.)
    copy_count: int                    # Количество копий
    additional_options: List[str]      # Доп. опции (рамка, ламинирование и т.д.)
```

#### Методы

---

##### `upload() -> str`

Фиксирует загрузку фотографии. Проверяет корректность количества копий.

**Исключение:** `ValueError` — если `copy_count <= 0`.

**Возвращает:** строку-подтверждение вида `"Фото P1 загружено"`.

```python
photo = Photo("P1", "10x15", 3)
photo.upload()   # -> "Фото P1 загружено"

bad = Photo("P2", "10x15", 0)
bad.upload()     # -> ValueError
```

---

##### `delete() -> str`

Фиксирует удаление фотографии.

**Возвращает:** строку-подтверждение вида `"Фото P1 удалено"`.

```python
photo.delete()  # -> "Фото P1 удалено"
```

### `Client`

Клиент фотоцентра.

```python
@dataclass
class Client:
    id: str      # Уникальный идентификатор
    name: str    # Полное имя
    phone: str   # Номер телефона
    email: str   # Адрес электронной почты
```

#### Методы

---

##### `validate() -> bool`

Проверяет, что обязательные поля (`name`, `phone`) не пустые.

**Возвращает:** `True` — если данные корректны, иначе `False`.

```python
c = Client("C1", "Иван Иванов", "+79001112233", "ivan@mail.ru")
c.validate()  # -> True

bad = Client("C2", "  ", "", "")
bad.validate()  # -> False
```

---

##### `receive_product(order_id: str) -> str`

Формирует подтверждение получения заказа клиентом.

| Параметр | Тип | Описание |
|---|---|---|
| `order_id` | `str` | Номер заказа |

**Возвращает:** строку вида `"Клиент Иван Иванов получил заказ О-001"`.

```python
c.receive_product("О-001")
# -> "Клиент Иван Иванов получил заказ О-001"
```

### `Order`

Заказ на печать фотографий.

```python
@dataclass
class Order:
    order_number: str        # Номер заказа (например "О-4A2B3C")
    order_date: str          # Дата создания в формате "дд.мм.гггг"
    client_id: str           # ID клиента-владельца заказа
    photos: List[Photo]      # Список позиций заказа
    status: str              # Текущий статус (по умолчанию "Новый")
    total: float             # Итоговая сумма (по умолчанию 0.0)
```

Допустимые статусы: `"Новый"`, `"В обработке"`, `"Готов"`, `"Оплачен"`, `"Выдан"`.

#### Методы

---

##### `calculate_cost(price_list: dict) -> float`

Рассчитывает итоговую стоимость заказа по всем позициям.
Обновляет атрибут `total` и возвращает его значение.
Форматы, отсутствующие в `price_list`, учитываются как 0 руб.

| Параметр | Тип | Описание |
|---|---|---|
| `price_list` | `dict[str, float]` | Словарь `{формат: цена}` |

**Возвращает:** суммарную стоимость заказа (`float`).

```python
photos = [Photo("P1", "10x15", 2), Photo("P2", "A4", 1)]
order = Order("О-001", "01.01.2026", "C1", photos)
prices = {"10x15": 15.0, "A4": 60.0}

order.calculate_cost(prices)  # -> 90.0  (2*15 + 1*60)
order.total                   # -> 90.0
```

---

##### `change_status(new_status: str) -> None`

Изменяет статус заказа на указанный.

| Параметр | Тип | Описание |
|---|---|---|
| `new_status` | `str` | Новый статус из списка допустимых |

**Исключение:** `ValueError` — если статус не входит в допустимые значения.

```python
order.change_status("В обработке")   # OK
order.change_status("Удалён")        # -> ValueError
```

### `Payment`

Платёж по заказу.

```python
@dataclass
class Payment:
    id: str               # Идентификатор платежа (например "П-4A2B3C")
    order_number: str     # Номер заказа
    payment_method: str   # Способ оплаты
    amount: float         # Сумма платежа в рублях
    status: str           # Статус (по умолчанию "Ожидает оплаты")
```

Допустимые способы оплаты: `"Наличные"`, `"Банковская карта"`, `"Оплата онлайн"`.

#### Методы

---

##### `process() -> str`

Проводит оплату: устанавливает статус `"Оплачен"`.

**Возвращает:** строку `"Оплачен"`.

```python
pay = Payment("П-001", "О-001", "Наличные", 90.0)
pay.process()   # -> "Оплачен"
pay.status      # -> "Оплачен"
```

---

##### `get_receipt() -> str`

Формирует строку кассового чека.

**Возвращает:** строку вида `"Чек №П-001: заказ О-001, Наличные, 90.00 руб."`.

```python
pay.get_receipt()
# -> "Чек №П-001: заказ О-001, Наличные, 90.00 руб."
```

### `Employee`

Базовый класс сотрудника фотоцентра.

```python
class Employee:
    role: str        # Должность
    name: str        # Имя
    _logged_in: bool # Флаг активной сессии (приватный)
```

#### Методы

---

##### `login() -> str`

Выполняет вход в систему, устанавливает флаг `_logged_in = True`.

**Возвращает:** строку вида `"Мария вошёл в систему"`.

```python
emp = Employee("Оператор", "Мария")
emp.login()         # -> "Мария вошёл в систему"
emp.is_logged_in    # -> True
```

---

##### `logout() -> str`

Выполняет выход из системы, устанавливает флаг `_logged_in = False`.

**Возвращает:** строку вида `"Мария вышел из системы"`.

```python
emp.logout()        # -> "Мария вышел из системы"
emp.is_logged_in    # -> False
```

---

##### `is_logged_in` *(property)*

Возвращает `True`, если сотрудник авторизован в системе.

### `Administrator`

Наследует `Employee`. Управляет прайс-листом.

```python
class Administrator(Employee):
    id: str   # Идентификатор администратора
    # role всегда "Администратор"
```

#### Методы

---

##### `set_price(price_list: PriceList, price: float) -> None`

Устанавливает новую цену в переданном объекте прайс-листа.

| Параметр | Тип | Описание |
|---|---|---|
| `price_list` | `PriceList` | Объект прайс-листа |
| `price` | `float` | Новая цена в рублях |

**Исключение:** `ValueError` — если `price <= 0` (выбрасывается внутри `PriceList`).

```python
admin = Administrator("A1", "Анна")
pl = PriceList("15x20", 25.0)
admin.set_price(pl, 30.0)
pl.get_price()  # -> 30.0
```

---

##### `edit_price(price_list: PriceList, new_price: float) -> None`

Редактирует существующую цену. Функционально идентичен `set_price`.
Семантическое разграничение: `set_price` — первоначальная установка, `edit_price` — изменение.

```python
admin.edit_price(pl, 35.0)
pl.get_price()  # -> 35.0
```

### `Cashier`

Наследует `Employee`. Проверяет оплату и выдаёт заказы.

```python
class Cashier(Employee):
    id: str   # Идентификатор кассира
    # role всегда "Кассир"
```

#### Методы

---

##### `verify_payment(payment: Payment) -> bool`

Проверяет, оплачен ли заказ.

| Параметр | Тип | Описание |
|---|---|---|
| `payment` | `Payment` | Объект платежа |

**Возвращает:** `True` если `payment.status == "Оплачен"`, иначе `False`.

```python
cashier = Cashier("K1", "Борис")

paid = Payment("П-001", "О-001", "Карта", 90.0, status="Оплачен")
cashier.verify_payment(paid)   # -> True

unpaid = Payment("П-002", "О-002", "Карта", 50.0)
cashier.verify_payment(unpaid) # -> False
```

---

##### `hand_out_product(order: Order) -> str`

Выдаёт заказ клиенту, если он оплачен. Меняет статус заказа на `"Выдан"`.

| Параметр | Тип | Описание |
|---|---|---|
| `order` | `Order` | Объект заказа |

**Возвращает:**
- `"Заказ О-001 выдан клиенту"` — если заказ оплачен;
- `"Заказ ещё не оплачен"` — если статус отличается от `"Оплачен"`.

```python
order = Order("О-003", "01.01.2026", "C1", status="Оплачен")
cashier.hand_out_product(order)   # -> "Заказ О-003 выдан клиенту"
order.status                      # -> "Выдан"
```

## Модуль `database.py`

Персистентное хранилище на основе JSON-файла. Содержит единственный класс `Database`.

---

### `Database`

```python
class Database:
    filepath: str   # Путь к JSON-файлу (по умолчанию "data.json")
```

При инициализации загружает данные из файла. Если файл не существует или повреждён — использует значения по умолчанию (встроенный прайс-лист).

#### Конструктор

```python
db = Database()                  # файл data.json в текущей директории
db = Database("my_data.json")    # кастомный путь (удобно для тестов)
```

---

#### `save() -> None`

Сохраняет текущее состояние всех данных в JSON-файл.
Вызывается автоматически всеми методами-мутаторами.

---

#### Методы работы с клиентами

##### `get_clients() -> dict`

Возвращает копию словаря всех клиентов `{client_id: {name, phone, email}}`.

##### `add_client(client_id: str, client_data: dict) -> None`

Добавляет нового клиента и сохраняет файл.

##### `delete_client(client_id: str) -> None`

Удаляет клиента по ID. Не выбрасывает исключение при отсутствии ID.

```python
db.add_client("C1", {"name": "Иван", "phone": "999", "email": "i@mail.ru"})
db.get_clients()   # -> {"C1": {"name": "Иван", ...}}
db.delete_client("C1")
db.get_clients()   # -> {}
```

---

#### Методы работы с заказами

##### `get_orders() -> dict`

Возвращает копию словаря всех заказов.

##### `add_order(order_id: str, order_data: dict) -> None`

Добавляет новый заказ и сохраняет файл.

##### `update_order(order_id: str, order_data: dict) -> None`

Обновляет данные существующего заказа и сохраняет файл.

```python
db.add_order("О-001", {"status": "Новый", "total": 90.0, ...})
db.update_order("О-001", {"status": "Оплачен", "total": 90.0, ...})
```

---

#### Методы работы с платежами

##### `get_payments() -> dict`

Возвращает копию словаря всех платежей.

##### `add_payment(payment_id: str, payment_data: dict) -> None`

Добавляет платёж и сохраняет файл.

##### `update_payment(payment_id: str, payment_data: dict) -> None`

Обновляет данные платежа и сохраняет файл.

---

#### Методы работы с прайс-листом

##### `get_price_list() -> dict`

Возвращает копию словаря `{формат: цена}`.

##### `set_price(format_name: str, price: float) -> None`

Добавляет новый формат или обновляет цену существующего.

##### `delete_price(format_name: str) -> None`

Удаляет формат из прайс-листа. Не выбрасывает исключение при отсутствии ключа.

```python
db.set_price("13x18", 20.0)
db.get_price_list()     # -> {..., "13x18": 20.0}
db.delete_price("13x18")
```
